# 헬스 커뮤니티 & 실시간 채팅 웹 서비스

## 프로젝트 소개

React + Vite 기반으로 제작된 헬스 파트너 매칭 웹 서비스이다.

사용자들은 자신의 운동 능력과 운동 정보를 프로필에 작성하여 서로를 확인할 수 있으며, 원하는 시간대와 장소 등을 협의하고 실시간 채팅을 할 수 있는 구조로 구성되어 있다.

로그인 방식을 구현할 때에는 JWT(JSON Web Token) 방식을 사용하였다. 로그인 인증 방식을 명확하게 지정해야 원하는 구조로 구현할 수 있을 것 같아 여러 인증 방식을 찾아보았고, JWT는 서버가 상태를 저장하지 않는 인증 방식으로 확장성이 높고 API 기반 서비스에 적합하다는 점 때문에 선택하였다.

사용자는 회원가입 및 로그인 이후 웹사이트의 기능을 이용할 수 있으며, 로그인에 성공하면 발급된 JWT 토큰을 통해 인증된 상태로 서비스를 사용할 수 있도록 구현하였다.

로그인 이후에는 마이페이지에서 본인 위치 인증을 통해 근처 이용자들과 매칭될 수 있도록 하였다. 위치 기반 기능 구현을 위해 카카오 지도 API와 네이버 지도 API를 사용하였다. 카카오 지도 API는 한국 지역 데이터의 정확도가 높고 무료 사용 범위가 충분하다고 판단하여 선택하였다.

사용자는 본인 위치 주변의 헬스장을 검색하고 선택하여 등록한 이후 다른 서비스를 이용할 수 있다.

게시글을 작성할 수 있으며, 다른 사용자가 작성한 게시글에 댓글을 달거나 운동 신청을 보낼 수 있다. 식단, 헬스 등 다양한 게시판을 구성하였으며, 최신순이나 나와 운동 중량이 비슷한 순 등 원하는 조건에 따라 게시글 정렬이 가능하도록 구현하였다.

마이페이지에서는 프로필 사진, 닉네임, 운동 경력 등 다양한 개인 정보를 수정할 수 있도록 하였다.

운동 신청을 보내면 상대방에게 신청 메시지가 전달되며, 상대방이 수락하면 자동으로 채팅방이 생성되어 두 사용자가 입장할 수 있다. 채팅 기능에는 읽음/안 읽음 표시 기능을 구현하여 사용자 간 상호작용이 가능하도록 하였다.

또한 운동 약속을 잡을 수 있으며, 운동 완료 버튼을 통해 실제 운동이 진행되었는지 확인할 수 있도록 하였다. 양측 모두가 운동 완료 버튼을 누르면 운동이 완료된 것으로 인정된다.

노쇼(No-show) 발생에 대비하여 신고 기능을 구현하였으며, 신고를 받을 경우 인기도가 15점 감소하도록 설계하였다. 반대로 운동을 정상적으로 완료하거나 후기를 받을 경우 인기도가 상승하도록 구현하였다.

인기도 시스템을 통해 사용자는 상대방의 프로필을 확인하고 인기도와 후기를 참고하여 비매너 이용자를 걸러낼 수 있도록 하였다.

채팅방 나가기를 통해서 채팅방에서 나올 수 있고, 서로에게서 채팅방이 사라질 수 있도록 하였다.

## 사용자 프로필

사용자는 자신의 운동 정보를 등록하고 수정할 수 있다.

- 닉네임  
- 프로필 아바타
- 다니는 헬스장
- 운동 경력  
- 벤치프레스  
- 스쿼트  
- 데드리프트  
- 3대 합계 자동 계산  
- 리뷰 확인

- 3대 기록  
- 리뷰 목록

## API

- `GET /api/auth/me` : 내 정보 조회  
- `PUT /api/users/:id` : 프로필 수정  
- `GET /api/users/:userId/profile` : 다른 유저 프로필 조회  
- `GET /api/chats/rooms` : 채팅방 목록 조회  
- `POST /api/chats/rooms/:roomId/leave` : 채팅방 나가기
