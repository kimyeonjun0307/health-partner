# 헬스 커뮤니티 & 실시간 채팅 웹 서비스

## 프로젝트 소개

React + Vite 기반으로 제작된 헬스 파트너 매칭 웹 서비스이다.  
사용자들이 운동 능력을 적은 프로필을 통해 서로를 확인하며, 원하는 시간대와 장소 등을 협의 하고, 실시간 채팅을 할 수 있는 구조로 구성되어 있다.

로그인 방식을 구현할 때에는 JWT방식을 사용했다. 로그인 인증 방식 역시 제대로 지정을 해줘야 의도대로 코딩을 해줄 것 같아서 찾아본 바로 
JWT는 서버가 상태를 저장하지 않는 인증 방식으로, 확장성과 API 구조에 적합하다고 하여서 JWT방식으로 로그인을 구현해달라고 했다.

사용자는 로그인 이후에 웹 사이트를 이용할 수 있다.

로그인 이후에는 본인 위치를 통해 근처 이용자들과 매칭이 될 수 있도록 위치 정보를 이용하려했다. 
카카오 지도 API와 네이버 지도 API를 썼는데, 카카오 지도 API는 한국 지역 데이터 정확도가 높고 사용이 무료 범위 내에서 충분하기 때문에 선택했다.

본인의 위치 주변에 헬스장을 등록한 이후에 다른 서비스를 이용할 수 있다.

글을 올릴 수도 있고 사람들이 올린글에 댓글, 요청등을 할 수 있다. 식단, 헬스, 등등 다양한 게시판이 있고
최신순,나와 중량이 비슷한 순,등등 원하는 조건에 따라 글 순서를 바꿀 수 있다.

신청을 하면 상대에게 신청메시지가 가고 상대가 수락하면 채팅방에 서로가 들어와진다.
거기서 대화를 할 수 있고, 대화는 상대가 읽음 안읽음과 같이 상호작용 기능을 구현했다.
약속을 잡을 수 있으며 운동 완료 버튼이 있어 서로가 완료 버튼을 눌렀을때 운동이 끝났다고 인정된다.
노쇼를 했을 경우를 대비해 신고 기능이 있고, 이는 인기도를 15점 내린다.
완료시 인기도가 오르고, 후기를 통해 또 인기도가 오를 수 있다.

인기도 시스템을 구현해놓아서 상대방의 인기도를 통해 비매너 이용자들을 최대한 거를 수 있게 노력했다.

인기도가 오르면 호칭도 바꾸는 시스템을 구현했는데, 마이페이지에서 보이지 않는 오류가 있어 구현해보려 노력했으나 구글 계정 5개와 커서 ide까지 썼음에도 끝내 
시간에 쫓겨 구현하지 못했다.

http://localhost:5173 웹사이트 
http://localhost:3000/ 서버주소.(웹사이트 아님)

아쉬운점: 하드코딩으로 만들어져서 인지 로컬환경에서만 열리고 서버에서는 열리지 않음. 열리더라도 서버와 웹사이트가 연동 되지 않아서 로그인이 되지 않음.
차후 서버에 대해서 더 공부를 하고 싶은 탐구욕이 생겼고, 다시 돌아와서 서버가 연동 되지 않았던 이유를 알고 해결해보고 싶다.

## 사용자 프로필

사용자는 자신의 운동 정보를 등록하고 수정할 수 있다.

- 닉네임  
- 프로필 아바타
- 다니는 헬스장
- 운동 경력  
- 벤치프레스  
- 스쿼트  
- 데드리프트  
- 3대 합계 자동 계산  
- 리뷰 확인  

---

## 프로필 조회 (모달)

다른 사용자를 클릭하면 상세 프로필이 모달로 표시된다.

- 닉네임, 아바타  
- 운동 경력  
- 3대 기록  
- 리뷰 목록  

---

## 실시간 채팅

1:1 채팅 기능

- 실시간 메시지 송수신  
- 채팅방 나가기

- 양쪽 모두 나가면 채팅방 삭제  
- 시스템 메시지 표시

## API

- `GET /api/auth/me` : 내 정보 조회  
- `PUT /api/users/:id` : 프로필 수정  
- `GET /api/users/:userId/profile` : 다른 유저 프로필 조회  
- `GET /api/chats/rooms` : 채팅방 목록 조회  
- `POST /api/chats/rooms/:roomId/leave` : 채팅방 나가기
